# de-ai：中文去 AI 味

把一段中文改得不像 LLM 写的。完整的模式目录、修复对照表、24 个案例在 `references/de-ai-tics-zh.md`——**改一段普通文本时不必整篇读**，按下面的流程走，命中可疑处再去查那一类的细则。

## 一条总原则

**让事实自己说话。** AI 味的根源不是用词华丽，是句子在**描述内容的形状**（"两半""三个层面""一句说明+三条动作"）、在**预报自己要说什么**（"下面把 X 拆开看"）、在**替读者下结论**（幻灯片底部的判定块）、在**给平淡内容贴情绪**（"三个问题躲不开"）。改法永远是：删掉外壳，留下事实。

## 三条诊断测试（核心工具）

判断一句话/一个词是不是 AI 味，跑这三个测试，比记句式表更可靠：

1. **删除测试**：把这句话（或这个词、这个"我"）删掉，信息少不少？
   - 不少 → 它是外壳，删。
   - 少了 → 它在传递内容，保留。
2. **换题测试**：把这句话换一个完全不相干的主题（比如换成做菜），它还成立吗？
   - 还成立 → 它在描述结构不是内容（"N 个要点 + M 个例子" 放哪都成立），重写成具体内容。
3. **叠加测试**：一个引子句里叠了几种 LLM 句式（时间锚点 / 计数 / 软动词 / 条件让步 / 元叙述）？
   - ≥ 3 种 → 整句重写或删除。真人一句话通常只用一种。

## 改写流程

### 第 0 步：分体裁

- **幻灯片**：最严。结论、过渡、解释都不上 slide，留给讲者口头说。底部判定块（alertblock）默认删。
- **公众号 / 推文 / 博客**：去掉导游词、垫话、反差金句、冗余第一人称。
- **报告 / 论文**：相对宽松，但仍去营销腔、空洞抽象动词、被动名词化。

### 第 1 步：扫结构层（最显眼，先抓）

- **判定块 / verdict block**：把刚展示过的东西再总结一遍的"核心问题/为什么算 X/这意味着"。→ 删。
- **元叙述 / 导游词**："下面把 X 拆开看""接下来我们来看""借着这次 X，N 个 Y 想聊""提一句先"。→ 删，直接进内容。
- **标题尾巴**："……你可以直接 clone 的""……几分钟就出稿"。→ 砍掉冒号后半截。
- **戏剧化容器名 / 引导词**："失败相册""三个问题躲不开"。→ 换成"N 个 X" 中性引导或直接列。

### 第 2 步：扫句式层（对照 references 第 1 节 1.1–1.9）

- 修辞对称："越…越…""不是 X 而是 Y""从 X 到 Y"（含短压缩版）。堆叠 → 留一处其余改直陈。
- 被动 + 名词化："被组织起来""AI 的使用""可 X 性"。→ 动词化。
- 教科书残留："知道 X 和 Y 的差别""X 的核心是""X 是一种 Y"。
- climax 强调："真正的 X""X 本身""其实就是""修辞性破折号 ——"。
- "可 X、可 Y、可 Z"三连 → 留最关键一个，其余换具体动词。
- 抽象动词："形成闭环/沉淀/落地/对齐/翻译成"→ "跑通/整理成/用上/按…来"。
- 营销腔："替你查/帮你管/让你不用/一键/直接套着用"→ 砍掉"替帮让"+第二人称、砍掉便利副词。
- hedge："某种意义上/归根结底/本质上/可以理解成"→ 多半直接删。
- 结构化容器："两半/两面/三块/N 个层面/N 维度"→ 自然量词或冒号直接列。

### 第 3 步：扫词汇层

对照 `references/de-ai-tics-zh.md` 第 2 节"高频修复对照表"逐词替换。

### 第 4 步：扫第一人称（散文尤其）

冗余的"我后来回头看/我最欣赏/我自己 X/比我自己以为的/因为我在 X 里写了"。用删除测试：删掉"我"句子还成立、信息没少 → 删或改被动。保留真实动作主语、反思核心、动机说明里的"我"。

### 第 5 步：输出

给两样东西：
1. **改写稿**（直接能用的版本）。
2. **修改对照**（改了哪几处、各属哪类、为什么）——这样用户能复核，也能学到模式。

## 重要提醒

- **压缩 ≠ 去除**。把"从 A 到 B 到 C 最后到 D"删成"从 A 到 D"还是同一个句式骨架。判断看骨架，不看字数。
- **改完先用三条测试自检一遍**，尤其是你自己写的总结语言——描述改动时很容易又用上"N 个 X + M 个 Y"这种模板。
- 发现新的 AI 味模式时，按 `references/de-ai-tics-zh.md` 第 5 节的模板追加一个案例。
