# 去 AI 味:中文表述中的 LLM 套路

> **用途**:作为一个 Skill 的训练 / 提示词素材,用来识别并改写 LLM 风格的中文文本(尤其是讲座、报告、幻灯片这类口语化体裁)。
>
> **维护**:基于工作流讲座修订过程中陆续指出的反面案例 + 提取的共性模式。每次发现新模式都追加在第 4 节。

---

## 0. 总原则

### 0.1 幻灯片是显示用的,不是论文

不要把需要口头表达的观点和结论直接写在幻灯片上,**除非这个结论非常重要**(讲座主线 / 听众必须带走的 1-2 个判断)。

幻灯片应当:
- **陈列**(摆出来让人看)
- **不要总结**(把"所以..."留给讲者口头说)
- **不要解释**(把"为什么..."留给图本身解释)

> 来自用户(2026-05):"我会在讲的时候说……这些话不会完整出现在幻灯片上。"
> 来自用户(2026-05):"结论应该由听众自行领会,除非这个结论是重大结论。"

#### 反模式:**"verdict block"(底部判定块)**

幻灯片底部加一个 alertblock 把刚才视觉/列表里展示的事**再总结一遍**——非常 AI。

**典型形态**(全部都是 LLM 标志):

| alertblock 标题 | 内容形态 |
|---|---|
| 为什么算"X" | 解释为什么前面的对照属于某个类别 |
| 核心问题 / 核心判断 | 重复或总结刚才视觉里的对比 |
| 这一章想说的 | 把章节内容预先告诉听众 |
| 启发 / 含义 / 这意味着 | 替听众做引申 |
| 要点 / 总结 / 关键在于 | 复述刚才的图 |

**修法**(默认):
1. **直接删 alertblock**(95% 的情况)
2. 如果删完整个 frame 太空,**把信息合并进 frame 标题**或**正文**,不要单独留 alertblock
3. **只在讲座主线的 1-2 个重大结论**才保留 alertblock(全场 60+ 帧里也就 2-3 帧应该有)

**判定标准**:这个 alertblock 删掉,听众看着上面的视觉/对照,能不能自己得出结论?
- **能** → 删
- **不能** → 视觉做得不够好,先去改视觉,不是补 alertblock 

**典型反例**(自己的"修复"先翻车的那种):
- ❌ Frame title "为什么这是一种研究基础设施" + alertblock "为什么算基础设施" 
- ✅ Frame title "传统 AI 使用 vs 研究 workflow"(直接对照,不下定语)

### 0.2 用具体名字,不用抽象类别

- ❌ "传统工具" → ✅ "Stata、Word"
- ❌ "研究判断" → ✅ "识别策略、解释、伦理"
- ❌ "执行性工作" → ✅ "数据清洗、写代码、排版"(只在引入时用抽象)

### 0.3 用动词,不用名词化

- ❌ "AI 的使用" → ✅ "用 AI"
- ❌ "X 的组织化" → ✅ "把 X 组织起来"
- ❌ "X 的差别" → ✅ "X 跟 Y 哪里不一样"

## 1. LLM 句式模式(按显眼程度排序)

### 1.1 ⚠️⚠️⚠️ 修辞性对称结构

最容易识别的 AI 味。出现一次还可忍,**堆叠多次就是死症**。

| 模式 | 例子 |
|---|---|
| 越...越... | "越往右,X 越被组织起来" |
| 不是 X,而是 Y | "不是要谁做的,而是这套过程能不能查"(单次保留;堆叠多次清掉) |
| 从 X 到 Y 到 Z,最后是 W | "从一句指令,到一份规则,到一套模块,最后是工作流" — 阶梯 climax |
| **从 X 到 Y**(短压缩版) | "用法从一句话到一整套" — **少几个台阶不算逃掉**,仍是同款模板 |
| 前 X%...后 X%... | "前 80% 的执行...后 20% 的判断" — 数字对称呼应(图里有就别在文字重复) |

### 1.2 ⚠️⚠️⚠️ 被动 + 名词化

中文不爱被动,LLM 写出来一堆。

| 模式 | 例子 |
|---|---|
| 被 X 起来 | "被组织起来"、"被翻译成"、"被沉淀下来" |
| X 的 Y(包动作成名词) | "AI 的使用"、"工作流的设计"、"知识的整理" |
| 可 X 性 | "可复查性"、"可追溯性"、"可执行性" |

### 1.3 ⚠️⚠️⚠️ 教科书 / 论文残留

学术写作的痕迹被带进口语场合。

| 模式 | 例子 |
|---|---|
| 学习目标语 | "知道 X 和 Y 的差别"、"理解 X 的本质"、"掌握 X" |
| 论文核心句 | "核心问题"、"核心判断"、"X 的核心是"、"X 的价值在于" |
| 定义句 | "X 是一种 Y"、"X 可以被定义为" |
| 章节过渡语 | "这一章想说的"、"接下来我们要"、"本节将" |

### 1.4 ⚠️⚠️ Climax 强调 tic

让一句话显得"重量"。

| 模式 | 例子 |
|---|---|
| 真正的 X | "真正的复现材料"、"真正要问的是" |
| X 本身 | "代码本身"、"这项研究本身" — 多半冗余 |
| 其实就是 X | "Gate 其实就是把 A 翻译成 B" |
| 修辞性 —— | "用一份 5 行的规则 —— 就能换掉 AI 的默认行为" |

### 1.5 ⚠️⚠️ "可 X、可 Y、可 Z" 三连堆叠

LLM 喜欢列形容词序列。

| 例子 |
|---|
| "可检查、可验证、可追溯的步骤" |
| "可执行证据 和 可复查记录" |
| "可查询、可复用、可追溯的研究知识库" |

**修法**:留 1 个最关键的,其他换成具体动词("能查"、"能跑回去看")。

### 1.6 ⚠️ 抽象动词

听起来高级,实际空洞。

| LLM 词 | 自然词 |
|---|---|
| 形成闭环 | 串成一个回路 / 跑通一遍 |
| 沉淀成 X | 整理成 X / 攒成 X |
| 落地 | 真的做出来 / 用上 |
| 对齐到 X | 按 X 来 / 跟 X 一致 |
| 翻译成 X | 换成 X / 改写成 X |
| 锚定 | 钉住 / 围绕 |

### 1.7 ⚠️ 同义并列三件套

三个抽象词排成一行,看着结构整齐,实际信息很少。

| 例子 |
|---|
| "重复劳动、工具调用、验证和记录" |
| "信息整理、知识沉淀、复用" |
| "拆解、执行、验证、留痕" |

**修法**:换成 2 个具体例子(具体压过抽象排比)。

### 1.8 ⚠️⚠️ 营销 / 推销话术

LLM 喜欢站在用户立场写"卖好"的话。出现在**标题尾巴**和**正文 bullet**都常见。

#### 1.8a 标题尾巴(promo tail)

| 模式 | 例子 |
|---|---|
| 你可以直接 X 的 Y | "开源 workflow 工具:你可以直接 clone 的" |
| X,几分钟就 Y | "AI 文献综述,几分钟就出稿" |
| 让你 Y 的 X | "让你少写一半代码的工具" |
| 一键 Y 的 X | "一键生成的综述脚本" |

**特征**:标题后半截带"你 / 让你 / 一键 / 几分钟"等读者承诺。

**修法**:**砍掉冒号后面那半截**。标题只保留客观主名词。

#### 1.8b 正文 bullet 里的"替你 / 帮你 / 让你"

| 模式 | 例子 | 修法 |
|---|---|---|
| 工具 X 替你 Y | "工具和验证器替你查" | "中间步骤都能查"(去掉"替你") |
| X 帮你 Y | "Skill 帮你管模板" | "Skill 管模板"(去掉"帮你") |
| 让你 X | "让你不用重写指令" | "不用重写指令"(去掉"让你") |

**特征**:bullet 里出现"替/帮/让"+ 第二人称(你),工具被拟人化成"替你做事的小弟"。

#### 1.8c "直接 / 一下 / 一键" 便利承诺

| 模式 | 例子 | 修法 |
|---|---|---|
| 直接 X 着用 | "下一次研究直接套着用" | "下一次还能用"(直说能用,不夸"直接") |
| 一键 X | "一键生成报告" | "生成报告"(去掉"一键") |
| 一下子就 X | "一下子就明白了" | "看一眼就明白" |

**共同特征**:用 adverb 强调"快 / 直接 / 不费力",感觉像广告语。

**修法**:**砍掉强调副词**,只留客观陈述。

### 1.9 ⚠️ Hedge / 缓冲词

让陈述显得"客观",实际是 LLM 不敢直说。

| 词 | 备注 |
|---|---|
| 某种意义上 | 几乎总能删 |
| 归根结底 | 几乎总能删 |
| 本质上 | 几乎总能删 |
| 可以理解成 | 改成 "等于" / "就是" |
| 可以视为 | 同上 |
| 已经不再是 / 不再只是 | 改成 "不是"(去掉"已经"、"只是") |

---

### 1.10 ⚠️⚠️⚠️ 自媒体标题党 / 舞台参与指令 / 比喻式辞令

讲座与幻灯片里最刺眼的一类,是"AI 帮忙写讲稿"的典型残留,一律清除。分四小类:

**(a) 自媒体标题党 / 短视频腔**(尤其是帧标题):
一页看懂、一图看懂、一句话读懂、X 页读懂、划重点、干货、硬核、秒懂、玩明白、上大分、破圈、天花板(作口号)、走 X 站 / 四站、一深四浅(X 深 Y 浅)、深讲、主菜、钩子、埋下悬念 / 先埋下 / 兑现悬念 / 埋下伏笔、揭晓。
→ 标题直陈其内容:"其余四篇简述""本节的四个部分""相似的研究""已见于研究文献的几个例子"。

**(b) 舞台参与指令**(对听众下的动作命令,幻灯片正文里不该出现):
全场起立、请起立、举手 / 请举手、与邻座交流 / 转向邻座、预测卡、请预测 / 先投一次票 / 猜一猜、现在举手 / 现在投票、花 60 秒、用一句话与…、闭上眼、想象你正…。
→ 改成客观陈述,或改成正式的"课后练习 / 思考题",或删除。

**(c) 比喻式 / 生造辞令**:
锚点(可信度锚点 / 能力锚点)、观察点、打开新数据源、同一条线上的延伸、落成(书面文档)、置信版本 /(含糊时的)质性版本、赋能、抓手、闭环(口号式)、拉满、颗粒度 / 对齐(作 buzzword)、生态位。
→ 用平实词:例证 / 结果 / 从图像提取数据 / 相似的研究 / 记录到 / 构建 / 按需。

**(d) 夸张收束金句**(帧末尾一句"点睛"):
"同一条流水线,替换…即可""这就是…的雏形""顶级公司""而非「可能」""已为我们探明""只学一件事""XX 越强,YY 越……"。
→ 删,或改客观陈述。

**自查三问**:① 帧标题是不是标题党? ② 有没有对听众的舞台指令? ③ 有没有比喻式生造词或帧末夸张金句?

---

### 1.11 ⚠️⚠️ 冗余元标注 / 自我加戏的解释(不节制)

幻灯片文字应当节制:让标题、图、列表自己说话,而不是把结构和理由都写出来、再重复一遍。这是典型 AI 表达——生怕读者看不懂,处处补注、处处解释。两小类:

**(a) 冗余元标注**(标题 / 小标题尾巴上再贴一个"这是几层 / 几步 / 几类"的结构标签,而正文已用 1/2/3 或分栏把结构讲明白):
:三层递进、:四步检查、:三个层级、:五个要点、(共 N 点)、总—分—总……
→ 删掉标签,标题只陈述内容。"从提示词到上下文与规则"已足够,不必再加":三层递进"。

**(b) 自我加戏的排序 / 理由**(列表项里塞进一句没人要的排序、强调或场景化理由):
"X 里,研究场景尤应先看 Y""其中最重要的是……""尤其要注意……""这一点尤为关键"。
→ 删。要强调,用加粗,不用整句旁白。

**自查**:这句话 / 这个标签删掉后,读者还能不能从标题、图、列表看懂?能,就删。

## 2. 高频修复对照表

| LLM 写法 | 自然写法 |
|---|---|
| 越往右,X 越被组织起来 | (删,让图自己说) / "用法从一句话到一整套" |

| 核心问题(alertblock 标题) | "想问的就一句" / 删 alertblock |
| 这一章想说的 | (整段删) / "这一章先看" |
| 真正的 X 是 Y | "X 就是 Y"(去掉"真正的") |
| X 本身 | (删) |
| 其实就是 X | "X" 直接定义 |
| 形成闭环 | "串起来" / "跑通" |
| 沉淀成 X | "整理成 X" |

| 让 X 变得 Y | "让 X 更 Y" / 去掉"变得" |

## 4. 案例存档(用户原话)

### 案例 1:"这一章想说的" 当 alertblock 标题
> "这一章想说的 这个表述太AI了 把这个都去掉"

**问题模式**:1.3 章节过渡语

### 案例 2:研究者的一天 — "核心问题" alertblock
> "这里的核心问题太AI味了 有什么改的方案"

**问题原文**:
> 【核心问题】前面这 80% 的执行性工作,能不能稳定交出去?这样研究者才有时间留给后面 20% 的判断。

**问题模式**:
- 1.3 alertblock 标题"核心问题"
- 1.1 数字对称(图里已有,文字又重复)
- "能不能...这样...才..." 修辞问 + 因果连接

### 案例 3:研究者的一天 — 不要直接表态
> "幻灯片不要把需要表达的观点和内容都直接表示出来,除非这个结论非常重要。"
> "我会在讲的时候说……这些话不会完整出现在幻灯片上。"

**问题模式**:0.1 总原则——把结论留给讲者口头说

### 案例 4:AI 使用层级 — "越往右..."
> "越往右,AI 的使用越被组织起来 这种表述太AI了"

**问题原文**:
> 越往右,AI 的使用越被组织起来:从一句指令,到一份规则、一套模块,最后是能反复跑、能复盘的工作流。

**问题模式**:
- 1.1 "越...越..." 对称
- 1.2 "AI 的使用" 名词化
- 1.2 "被组织起来" 被动 + nominalization
- 1.1 "从 X 到 Y,最后是 W" climax 阶梯
- "能 X、能 Y 的 W" 形容词模式

**最终改成**:"用法从一句话到一整套。"

### 案例 5:本讲内容 — "知道它和传统工具的差别"
> "知道它和传统工具的差别 这种表达也太AI了"

**问题模式**:
- 1.3 "知道 X 的 Y" 学习目标语
- 0.2 "传统工具" 抽象代称

**最终改成**:"跟 Stata、Word 哪里不一样"

---

### 案例 8:基础设施帧 — verdict block + 判定语气标题
> "在幻灯片的下面不要总是加'为什么算'基础设施''这种 verdict block,这个也很 AI,结论应该由听众自行领会,除非这个结论是重大结论。"

**问题原文**:
> Frame 标题:**为什么这是一种研究基础设施**
>
> 底部 alertblock 标题:**为什么算"基础设施"**
> 内容:重复劳动、工具调用、验证、记录这几件事一旦固定下来,下一个项目能直接接着用,不用从零搭。

**问题模式**:
- 0.1 反模式 **verdict block**(底部判定块,标题尤其爱用"为什么算 X")
- 0.1 frame title 本身就预下判定("为什么 X 是 Y")
- 双重判定:标题先下了结论,alertblock 又重复了一遍

**最终改成**:
- Frame title → **"传统 AI 使用 vs 研究 workflow"**(纯描述对照)
- 底部 alertblock → **整段删除**

**经验教训**(写入 0.1 总原则):
1. 默认假设 = "听众自己能看出来"
2. 重大结论(讲座主线、必须带走的 1-2 个判断)才保留 verdict block
3. 60+ 帧里如果出现 20+ 个 alertblock,基本说明在到处下判定

---

### 案例 9:研究 workflow bullet — "替你 X" + "直接套着用"
> "工具和验证器替你查 / 下一次研究直接套着用 这些表述有些 AI 味"

**问题原文**:
> 研究 workflow 列表 4 条 bullet:
> - 任务有步骤、有中间产物 ✓
> - **工具和验证器替你查** ← AI
> - 失败留下来,写进规则 ✓
> - **下一次研究直接套着用** ← AI

**问题模式**:
- 1.8b "X 替你 Y" — 工具被拟人化成"替你做事的小弟"(站在用户立场卖好)
- 1.8c "直接 X 着用" — 便利承诺/广告语,强调"不费力"

**最终改成**:
- "工具和验证器替你查" → **"中间步骤都能查"**
- "下一次研究直接套着用" → **"下一次还能用"**

### 案例 13:对话总结里的"一句说明 + 三条具体动作"(我自己又翻车)
> "一句说明 三条具体动作 这个AI味太浓了"

**背景**:这次出问题的**不是幻灯片**,而是我(Claude)在改完幻灯片之后,**对用户描述改动**时用的总结语言。

**问题原文**(我对话里的):
> 三栏每栏一个 block,每个 block:**"一句说明 + 三条具体动作"**

**问题模式**:**结构模板式描述**——用"数量+类别"的对仗骨架代替具体内容。
- 形态:"N 个 X + M 个 Y"、"一句话 + 一张图 + 一个例子"、"X 件事 + Y 个建议"、"总分总结构"
- 和 1.5 "可 X、可 Y、可 Z" 同一家族(抽象数量 + 抽象类别 + 对仗)
- 关键症状:把"说明"、"动作"这种**最高级别的抽象名词**拿出来对仗,完全不传递任何关于内容的信息

**应当怎么写**:用**具体内容**做描述,不用结构。

| 翻车版(模板) | 正确版(内容) |
|---|---|
| "一句说明 + 三条具体动作" | "披露要在论文里写什么 / 隐私怎么按敏感度分级 / 复现要存 prompt 和模型版本号" |
| "N 个要点 + M 个例子" | 直接列出几个要点是什么 |
| "总分总结构" | 直接说讲了什么 |

**经验教训(元层级)**:

去 AI 味不是只针对"被改的文本"——**我自己描述工作的语言**也是 AI 味重灾区。改 LLM 模板的同时,容易用 LLM 模板**汇报**改动。这是和案例 6 同类的二次翻车(那次是改完留下"从 X 到 Y"原句骨架;这次是用结构模板**总结**改动)。

**检查清单更新**:每次回复用户之前,扫一遍自己的总结语言,问:
1. 有没有"N 个 X + M 个 Y" 这类对仗骨架?
2. 有没有"说明 / 动作 / 维度 / 层面 / 角度"这类抽象类别词?
3. 我说的内容**换一个完全不同的主题**(比如换成做菜),这句话还成立吗? 如果还成立,说明我没在描述真东西,只在描述结构。

**新增模式(归入 1.x)**:
- **结构模板式描述**:"N 个 X + M 个 Y"、"一句话 + 一张图 + 一个例子"、"X 件事 + Y 个建议" —— 用抽象数量与抽象类别拼对仗,完全不传递内容信息。常见于汇报、总结、文档导读。**修复**:替换为对具体内容的概括(具体讲了什么 / 改了什么具体地方),不要用"说明 / 动作 / 维度 / 角度"这类零信息类别名。

### 案例 15:公众号文章 — "坦白讲——...心里都有数,后面会回来再说"
> "不过坦白讲——这个题目是我和 AI 一起编出来的,不算一个真问题。这一点我和 Paper Factory 心里都有数,后面会回来再说。 这个AI味太浓了,另外这里应该说明paper factory不负责选题"

**问题原文**:
> 不过坦白讲——**这个题目是我和 AI 一起编出来的,不算一个真问题。这一点我和 Paper Factory 心里都有数,后面会回来再说**。

**问题模式**(一句叠了三层 AI 味):

| 层 | 句段 | 类别 |
|---|---|---|
| 1 | "不过坦白讲——" | 1.9 hedge 软开头;confessional 自我修饰,装诚实 |
| 2 | "我和 Paper Factory 心里都有数" | **工具拟人化** + "心里有数" 口语套话;把工具当共谋者 |
| 3 | "后面会回来再说" | 案例 14 导游词,告诉读者"等下还要回到这" |

三连叠加更显矫情。原句**只 admit"题目不真"**,但**没承担本该承担的信息**——读者会问:那 Paper Factory 自己能不能选题?这事得写清楚。

**修复**(同时做两件事:**直接陈述工具边界 + 自陈题目来路**):
> **Paper Factory 不负责选研究问题——你给什么问题,它就跑什么问题。这个题目是我配着手头数据拼出来的,不算真问题,只是想让流水线先跑起来看看。**

变化:
- 去掉"坦白讲——"/"心里都有数"/"后面再说"三层软化
- 把工具的真实边界(不选题)放到句首陈述,**讲承担信息的话,不讲卖萌的话**
- 把自陈("我编的")放在 statement 之后,作为说明而不是悔过

**新增模式(归入 1.x)**:
- **工具拟人化共谋**:"我和 X 心里都有数 / 它和我都明白 / 它知道我在干什么" 等把工具说成"心知肚明的同谋"。**修复**:把工具当工具,事实陈述就好。
- **confessional 软开头**:"说实话 / 坦白讲 / 不瞒你说 / 老实说" 等装诚实的修辞性开场。**修复**:删掉,直接说事。如果真有要承认的事,陈述事本身就够诚实。
- **该承担信息的句子在卖萌**:写到一个关键 limitation 时,用"心里都有数 / 后面再说"等软化语回避正面陈述。**诊断标准**:这句话要传达的关键信息是什么?如果删掉所有软化语后那句信息还在,就保留;如果删掉后什么都没剩下,说明这句话本来就在卖萌不在传达。

### 案例 16:公众号文章 — 刻意强调自我的第一人称插入
> "我后来回头看,这个判断没什么毛病。 这个我写的很刻意,全文很多这种刻意强调自我的情况也不太符合语言表达习惯"

**问题原文**(典型一例):
> ...拿它替代,并在 `data_wrangle.md` 和 `data_context.md` 两份报告里把这个替代决策都写清楚了。**我后来回头看,这个判断没什么毛病。**

**问题模式**:**冗余第一人称强调** —— 在叙述里反复插入"我 / 我自己 / 我后来 X / 我事后看 / 比我自己之前以为的 / 我最欣赏的"等不传递新信息的第一人称片段。

类型清单:

| 类型 | 例子 | 问题 |
|---|---|---|
| 自我评价插入 | "我后来回头看,这个判断没什么毛病" | 整句是自我点评,没传递事实 |
| "我最 X 的一处" | "这是 Paper Factory 设计中我最欣赏的一处" | 把主观偏好强加到客观陈述上 |
| "我自己 X 的" | "和我自己在做的 skill 列出的反面模式重合" | "自己" 双重强调,冗余 |
| "比我自己之前以为的" | "比我自己之前以为的成熟很多" | "自己" 多余 |
| "我编出来的" 重复 | 前面已经交代过是作者编的,后面又说一遍 | 重复占位 |
| "因为我在 brief 里写了" | 把"brief 里写的"事实表述成"我做的事" | brief 当主语就够,我不必出现 |

**为什么是问题**:中文长文有一个写作直觉是**让事实自己说话**。叙述的主语已经清楚谁在写,事实陈述本身就够;反复加"我"是把读者注意力从"事情"拉回到"作者"。LLM 容易堆这种第一人称是因为它经常用"我观察到 / 我注意到 / 我认为"作填充,以及它输出"反思 / 自陈"段落时会过度自我标记。

**修复策略**:

1. **整句自我评价 → 客观结论**: "我后来回头看,这个判断没什么毛病" → "事后看是合理的处理"
2. **"我最 X 的一处" → 客观陈述**: "我最欣赏的一处" → "一个值得说的细节"
3. **删 "自己" 冗余强调**: "我自己在做的" → "我在做的";"比我自己之前以为的" → "比我之前以为的"
4. **把作者改成被动 / 让事物当主语**: "是因为我在 brief 里写了" → "是因为 brief 里就这么写了";"套到我给的数据上" → "套到给定的数据上";"Q2 那个题目是我编出来的" → "Q2 那个题目是配合数据编出来的"
5. **删整句**: "这正是我这次干的事" 这种回指作者意图的句子,如果上下文已经清楚,可以直接删

**保留 "我" 的合法场景**:
- 真实动作主语("我手动加了三个 symlink"——确实是亲手做的)
- 反思段第一人称("我以为...跑完之后,我修正了这个判断"——反思的核心)
- 试跑动机("我拿现成的数据让它跑了一遍"——讲为什么做)

**诊断标准**:把句子里的"我 / 我自己"删掉,如果**句子还自然成立、信息一点没少**,这个"我"就是冗余;反之就是必要的 actor marker。

**新增模式(归入 1.x)**:
- **冗余第一人称强调**:"我后来 X / 我事后看 / 我最欣赏 / 我自己 X / 比我自己以为的 / 因为我 Y" 等不传递新信息的第一人称插入。**修复**:删掉、换被动语态、或让事物当主语。**诊断**:"我"删掉后句子还成立、信息没少,就是冗余。

### 案例 17:公众号文章 — "原来的研究问题包含两半"
> "原来的研究问题包含两半 这个两半是AI表述"

**问题原文**:
> 原来的研究问题**包含两半**——质量-价格 grid 描述(Q2.1)+ 开源 release 的位移事件研究(Q2.2)。

**类型清单**:

| 模式 | 例子 | 自然版 |

| 包含两半 | "原来的研究问题包含两半" | "研究问题原本有两个" |
| 分为两面 | "这个工作流分为两面" | "这个工作流有两块" |
| 拆成三块 | "可以拆成三块来看" | "里面有三件事" (或者直接列) |
| 包含 N 个层面 | "包含三个层面的设计" | "里面有三层 / 三件设计" |
| N 维 / N 维度 | "从两个维度去看" | "从两边看 / 两件事看" |

**修复**:换成自然量词("两个 / 两件事 / 两块")或者**直接用冒号 / 顿号 / "和" 列出**, 不必先盖一个容器再装内容。

**修复后**:
> 研究问题**原本有两个**:质量-价格 grid 描述(Q2.1)**和**开源 release 的位移事件研究(Q2.2)。

**新增模式(归入 1.x)**:
- **结构化容器分割词**:"两半 / 两面 / 三块 / N 部分 / N 维度 / N 个层面"等抽象容器把内容硬塞进对称结构。和"结构模板式描述"同源——讲的是形状,不是内容。**修复**:换自然量词("两个 / 两件事")或直接用 ":" "和" 列出。**诊断**:把"两半 / 两面"删掉,改成"两个"或冒号列举,意思一点没变就是冗余。

### 案例 18:公众号文章 — "三个问题躲不开"
> "三个问题躲不开 这个很ai味"

**问题原文**:
> 看完它的产出,**三个问题躲不开**:

**问题模式**:**戏剧化列举引导词**——把"接下来我要列 N 件事"这件平淡事包装成"这 N 件事已经避不开了 / 浮现出来了 / 摆在桌面上了",给读者加情绪压力。

类型清单:

| 模式 | 例子 |
|---|---|
| X 躲不开 | "三个问题躲不开" |
| X 绕不过去 | "这几件事绕不过去" |
| 摆在桌面上 | "三个问题摆在桌面上" |
| X 浮现 | "三个问题浮现出来" |
| X 挥之不去 | "几个想法挥之不去" |
| X 不容回避 | "三件事不容回避" |
| 不得不问 | "不得不问的三个问题" |

**特征**:动词或形容词带"必须面对 / 无法回避"的情绪重量,但实际上下文只是要列举内容。和案例 14"导游词式元叙述"是近亲——区别在于导游词是平铺直叙的预报("下面讲三件事"),戏剧化引导词是情绪化的预报("三件事避不开")。

**修复**:删掉戏剧动词,让事物自己出现。

- "三个问题躲不开:" → "**有三个问题想问:**"
- "几件事绕不过去:" → "**有几件事:**"
- "三件事浮现出来:" → "**三件事:**"

或更直接:删整句引导,让加粗的问题或 bullet 自己开始。

**新增模式(归入 1.x)**:
- **戏剧化列举引导词**:"N 个 X 躲不开 / 绕不过去 / 摆在桌面上 / 浮现 / 挥之不去 / 不容回避 / 不得不 Y"等带情绪重量的列举开场。**修复**:删掉戏剧动词,改成中性引导("有 N 个 X:" / "N 件事:")或直接让事物自己出现。**诊断**:把动词删掉,引导句变成中性陈述,内容不少——动词就是戏剧化垫词。

### 案例 19:公众号文章 — "提一句先："
> "提一句先: 这个AI味很重"

**问题原文**(数据列表 + 引用块之间的衔接):
> - Epoch AI 模型属性表(3,237 个模型,含 open-weight 标记)
>
> **提一句先:**
>
> > **Paper Factory 不负责选研究问题——你给什么问题,它就跑什么问题。**

**问题模式**:**对话式垫话** —— 一句不传递任何内容、只宣告"我要说一句话"的引导。和案例 14(导游词)同族,但属于"对话式"亚种:不是预告整章/整段结构,而是给单个句子加挡板。

类型清单:

| 模式 | 例子 |
|---|---|
| 提一句先 | "提一句先:..." |
| 顺便说一句 | "顺便说一句..." |
| 先扯一句 | "先扯一句..." |
| 多说一句 | "多说一句..." |
| 说几句题外话 | "说几句题外话..." |
| 这里要插一句 | "这里要插一句..." |
| 先停一下 | "先停一下..." |
| 容我多说一句 | "容我多说一句..." |

**特征**:在散文里像礼貌的清嗓子。书面正式语境(论文、报告)可能合适,但**公众号 / 微博 / 推文里读者已经在读这一句**,不需要被告知"我要说一句话"。

**修复**:**整句删除**。如果接的下一句(比如引用块、加粗句)足够 standalone,删掉对话式垫话之后整段不会有缝隙。

**修复后**:
> - Epoch AI 模型属性表(3,237 个模型,含 open-weight 标记)
>
> > **Paper Factory 不负责选研究问题——你给什么问题,它就跑什么问题。**

**诊断标准**:把垫话句删掉,前后段落自己能接上 — 那这垫话就是冗余清嗓子。如果删掉之后出现逻辑断层,可能不是垫话,是真过渡句。

**新增模式(归入 1.x)**:
- **对话式垫话**:"提一句先 / 顺便说一句 / 先扯一句 / 多说一句 / 容我多说一句 / 这里要插一句 / 先停一下"等单句级 meta 宣告。**修复**:整句删除。**诊断**:删掉后前后还能接上 → 是垫话;接不上 → 是真过渡(需要换成内容性过渡,不是删除)。

### 案例 20:公众号文章 — "中间我只睡了一觉。"
> "中间我只睡了一觉。 这个太ai味了"

**问题原文**(开场段落收尾):
> 查文献、清数据、跑Stata、写LaTeX、复算数字、自己挑错——连去AI味都自动跑了一遍。
>
> 全程6小时32分钟。**中间我只睡了一觉。**

**问题模式**:**"长事实+短反差金句"的 LLM 收尾节奏**。

这种 punch line 句子有几个 LLM 偏好的特征叠在一起:

| 特征 | 体现 |
|---|---|
| 长句 + 短句反差 | 前句 "全程 6 小时 32 分钟" 数字陈述,后句"中间我只睡了一觉"短而轻 |
| 第一人称 + 限定词 | "我 + 只" 双标记,看似谦虚实则给"AI 干了全部"贴情绪标签 |
| 单独成段强调 | 后句独占一段,自带"截图金句"姿态 |
| 隐性自夸 | 表面"我没干啥",实际抬高 AI 的成就(我不在的情况下它都搞定了) |

这种节奏是 LLM 段落收尾的高频模板:**"X 件大事自己完成。中间我只 Y。"**

**为什么是问题**:它**没传递新信息**。前一句"连去 AI 味都自动跑了一遍"已经隐含"作者参与度极低"。后一句的"我只睡了一觉"只是把这点重新包装成幽默金句——而幽默的形状本身就是 LLM 模板。

**修复**:**整句删除**。让前一段 + "全程 6 小时 32 分钟" 单独收尾,把人的角色留作隐含信息。

**修复后**:
> 查文献、清数据、跑 Stata、写 LaTeX、复算数字、自己挑错——连去 AI 味都自动跑了一遍。
>
> 全程 6 小时 32 分钟。

**诊断标准**:

1. 这句话**删掉后信息是否不少**?(在这例里前一句已经隐含,所以信息没少)
2. 这句话的**节奏形状**是不是"长事实 + 短反差 + 第一人称 + 限定词"?
3. 它是不是想被截图当金句?

三个都中,就是 LLM 模板化金句,删掉。

**新增模式(归入 1.x)**:
- **反差金句节奏**:"X 件大事自己完成。中间我只 Y。" / "整个过程花了 X 小时。我只 Y。" / "AI 干了 X、Y、Z。我什么都没做。" 这一类"长事实 + 短反差金句"结构。**修复**:删整句。让事实自己说话,**不需要给"AI 自动化"贴情绪戏剧标签**。

### 案例 21:公众号文章 — "借着这次试跑,三个问题想聊:"
> "借着这次试跑,三个问题想聊: 这个表达也是ai味"

**问题原文**(承接段后引出三个 bold 问句):
> ...剩下的全是机器自己的事。
>
> **借着这次试跑,三个问题想聊:**
>
> **workflow 这个概念到底在说什么?**
> ...

**问题模式**:**"半 formal 引子 + 计数列举 + 软动词"的三联套**——这次是个稍微更高级的导游词变体。

三层叠加:

| 层 | 体现 |
|---|---|
| 半 formal 引子 | "借着/趁着/顺着/就着 + X" — 略书面、用动词把现状变成"机会" |
| 计数 + 软动词列举 | "N 个 X 想 Y" — 用"想聊/想说/想讲/想问"等暖男话术包装enumeration |
| 冒号引出 bullet 列表 | 经典 LLM 段落 layout |

类型清单:

| 模式 | 例子 |
|---|---|
| 借着 X,N 个 Y 想 Z | "借着这次试跑,三个问题想聊" |
| 趁着 X 的机会,N 件 Y 想 Z | "趁着这次机会,几件事想说" |
| 正好这次 X,N 个 Y 想 Z | "正好这次实践,几个话题想讲" |
| 顺着这条 X,N 个 Y 想 Z | "顺着这条线索,几个问题想聊" |
| 借此机会 X | "借此机会聊聊..." |

**本质问题**:还是**元叙述**——告诉读者"我接下来要讲 N 件事",而不是讲事本身。和案例 14、18、19 同族,只是包装更精致。

**修复策略**:换成**内容性过渡**——动词描述"事情留下了什么"或"状态如何",而不是"我要说什么"。

| 替换思路 | 例子 |
|---|---|
| 用"留下"描述经验后的状态 | "这一夜跑下来,留下三个问题:" |
| 用"剩下"描述未解的内容 | "跑完之后,剩下几件事没想通:" |
| 用具体时间锚点 + 名词 | "凌晨四点半 paper.tex 出来后,三个问题:" |
| 直接删除引子,让 bullets 自己出现 | (适合上下文已经足够明确的场景) |

**修复后**:
> 这一夜跑下来,留下三个问题:
>
> **workflow 这个概念到底在说什么?**
> ...

**新增模式(归入 1.x)**:
- **半 formal 计数引子三件套**:"借着 / 趁着 / 顺着 / 就着 / 正好 / 借此 + X + N 个 + Y + 想 + Z"——用书面动词把"现状变机会",再加计数 + 暖男化软动词列举。**修复**:换成内容性动词("留下 / 剩下 / 摆着"),让句子描述事实而不是宣告意图。

### 案例 22:公众号文章 — "试跑完一轮,有 5 条姿势不依赖 Paper Factory 也能往自己研究里搬。"
> "试跑完一轮,有 5 条姿势不依赖 Paper Factory 也能往自己研究里搬。 这个也很ai味"

**问题原文**(H2 标题下,五个 H3 之前的过渡):
> ## 学术研究能学走什么
>
> 试跑完一轮,有 5 条姿势不依赖 Paper Factory 也能往自己研究里搬。
>
> ### 一、让 AI 跑多版本...

**问题模式**:**"经历完 + 计数列举 + 条件让步"三件套**——前几个案例的复合升级版,几个独立看都不算重的 LLM 习惯句式叠在一起,合起来就明显:

| 成分 | 例子 | 单看的问题 |
|---|---|---|
| "X 完一轮 / X 完一遍 / X 下来" | "试跑完一轮" | 时间锚点句式,LLM 偏爱"经历完"叙事 |
| "有 N 条 X" | "有 5 条姿势" | 计数列举宣告(同案例 21) |
| "不依赖 Y 也能 Z" | "不依赖 Paper Factory 也能往自己研究里搬" | 条件让步小句,LLM 喜欢的"既要又要"包装 |
| 隐藏第四层:整句是元叙述 | 整体在宣告"下面要讲 5 件事" | 同案例 14、18、19、21 |

**为什么单看不重、叠在一起就 AI**:每一种 LLM 句式都有自然中文的合理用法。问题是 LLM 倾向于**在一句话里叠三种以上**,营造"内容丰富又克制"的错觉。真人写中文,一句话通常只会用其中一种。

**修复策略**:**整句删除**。如果 H2 标题 + H3 编号已经给出列举信号,中间这一行的工作纯冗余。

**修复后**:
> ## 学术研究能学走什么
>
> ### 一、让 AI 跑多版本...

H2 标题"学术研究能学走什么"问了一个问题,H3 "一、X" 是答案的第一条。中间不需要"下面有 N 条答案"的导游词。

**新增模式(归入 1.x)**:
- **多句式叠加的元叙述**:单看每个句式都不算重(时间锚点 + 计数 + 条件让步 + 隐含元叙述),但**两三种叠在一起就是 LLM 标志性的"包装克制感"**。**诊断**:数一下一个引子句里有几种 LLM 常见句式 ≥ 3 种就重新写或直接删。**修复**:删整句或拆成多个短句,每句只承担一种功能。
