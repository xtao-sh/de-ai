# de-ai

中文去 AI 味 Codex skill。它用于审阅和改写中文文本，让幻灯片、讲稿、报告、公众号文章里的 LLM 腔更少，表达更具体、更自然。

## 适用场景

- 中文幻灯片：删掉底部判定块、导游词、过度总结和不该写在页面上的讲稿。
- 公众号、推文、博客：删掉垫话、反差金句、营销腔和冗余第一人称。
- 报告、论文草稿：降低名词化、被动句、空洞抽象动词和模板化过渡。

## 核心方法

这个 skill 用三条测试判断一句话是否有 AI 味：

1. 删除测试：删掉以后信息少不少？
2. 换题测试：换成别的主题以后句子还成立吗？
3. 叠加测试：一句里是否叠了时间锚点、计数、软动词、让步、元叙述等多个模板？

原则很简单：让事实自己说话。少写结构说明、预告、判定和情绪包装。

## 目录结构

```text
de-ai/
  SKILL.md
  README.md
  LICENSE
  references/
    de-ai-tics-zh.md
```

`SKILL.md` 是入口文件。`references/de-ai-tics-zh.md` 保存完整规则、修复对照表和匿名化修订案例。

## 安装

从 GitHub 安装：

```bash
npx skills add https://github.com/xtao-sh/de-ai --skill de-ai
```

列出仓库里的 skill：

```bash
npx skills add https://github.com/xtao-sh/de-ai --list
```

安装后重启 Codex。

## 使用示例

```text
Use $de-ai 帮我改一下这段公众号文字，去掉 AI 味，保留原意。
```

```text
Use $de-ai 检查这几页幻灯片，删掉不该写在 slide 上的总结句和导游词。
```

```text
Use $de-ai 这段太像机器写的，给我一版自然中文，并列出修改对照。
```

## 输出格式

默认输出两部分：

1. 改写稿：可以直接使用的版本。
2. 修改对照：说明改了哪些句子、属于哪类问题、为什么这样改。

## 维护规则

新增案例时，追加到 `references/de-ai-tics-zh.md` 第 5 节模板中：

```markdown
### 案例 N: [简短标题]
> "[匿名化反馈]"

**问题原文**: [出问题的文字]

**问题模式**: [匹配到哪一类模式]

**最终改成**: [修正后的文字]
```

发布前请检查：

- 不包含真实人名、账号、邮箱、电话或私人项目路径。
- 引用文件在 skill 目录内，不依赖本地软链接。
- 案例已匿名化，不暴露私人对话上下文。
- 不把它描述成 AI 检测或规避检测工具；它是中文写作和编辑规则。

## 隐私

这个 skill 只提供文本审阅和改写规则，不需要读取本地凭据，也不需要访问外部服务。公开发布前仍应检查案例文本，避免包含私人信息。
